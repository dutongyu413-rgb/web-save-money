# 反向记账 UI 视觉规范（MVP）

> 适用范围：React + TypeScript Web App，以及后续小程序内嵌页面。

## 1. 视觉定位

产品定位为轻量、清晰、容易长期使用的个人记账工具。界面不追求传统金融 APP 的厚重感，也不使用大面积品牌色，重点突出收入、净储蓄和储蓄率。

设计原则：

- 浅灰画布承托内容，页面保持通透。
- 白色卡片只用于需要明确分组的数据。
- 蓝色表示收入、主要操作和可点击状态。
- 橙色表示净储蓄、储蓄率和目标完成进度。
- 红色仅用于删除、清空和覆盖恢复等危险操作。
- 首页第一视觉焦点始终是储蓄率圆环。

## 2. 色彩规范

| Token | 色值 | 用途 |
| --- | --- | --- |
| `blue-700` | `#465D75` | 收入深色文字和次级操作 |
| `blue-600` | `#617891` | 收入数据、图表和图标 |
| `blue-050` | `#EDF2F7` | 收入图标浅色背景 |
| `orange-700` | `#DF5900` | 净储蓄与储蓄率文字 |
| `orange-500` | `#F56A0A` | 储蓄按钮、图表和进度 |
| `orange-050` | `#FFF3E9` | 储蓄提示背景 |
| `ink-900` | `#151719` | 标题和正文 |
| `ink-600` | `#686B70` | 辅助说明 |
| `ink-500` | `#7A7D82` | 图表坐标轴与弱提示 |
| `line` | `#E7E7E3` | 分割线和输入框边界 |

| `danger` | `#BB3D4A` | 危险操作 |

页面标题区不得使用大面积蓝色背景。蓝色和橙色只作为数据与操作强调色。

## 3. 字体与数字

字体使用系统字体，正文优先使用系统文本字体：

```css
-apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC", "Microsoft YaHei", sans-serif
```

- 首页标题：24px / 700。
- 二级页面标题：21px / 700。
- 区块标题：17px / 700。
- 核心百分比：42–52px / 600。
- 关键金额：18–31px / 600。
- 正文与列表标题：14px / 500–600。
- 标签与辅助说明：12–13px / 400–600。
- 图表坐标轴和图例：不小于 11px。

金额和百分比使用等宽数字特性，减少数字变化造成的视觉跳动。

## 4. 间距与形状

- 页面左右边距：18px；小于等于 360px 时为 16px。
- 内容最大宽度：480px。
- 首页核心卡片圆角：28px。
- 普通卡片圆角：20px。
- 输入框圆角：16px。
- 胶囊圆角只用于主要按钮和切换控件。
- 触控区域不小于 44 × 44px。
- 核心卡片阴影：`0 10px 30px rgba(22, 24, 26, 0.055)`。
- 普通卡片优先只使用边框；需要悬浮感时使用更轻的次级阴影。

卡片阴影保持非常轻，不使用厚重投影、渐变背景和玻璃拟态。

## 5. 页面规范

### 5.1 首页

页面结构固定为：

1. 产品标题。
2. 月份切换。
3. 包含“这个月，你存下了多少？”、储蓄率圆环、收入、净储蓄和目标进度的核心大卡片。
4. 记收入、记储蓄两个操作按钮。
5. 必要时显示未记录储蓄的轻提示。

未来月份不可进入；历史月份显示“回到本月”，且不能造成月份标题位移。总支出在 MVP 首页不展示。储蓄超过目标时只显示“已达标”。

### 5.2 收入和储蓄记录页

- 顶部仅保留返回和页面标题，不显示全局导航。
- 累计汇总使用白色卡片，不使用整块彩色背景。
- 收入金额用蓝色，净储蓄金额用深橙色。
- 展示全部历史记录，并按月份分组，不提供月份切换。
- 记录列表使用白色面板、轻分割线且不使用投影。
- 每页只保留底部“记录一笔”按钮，不在右上角重复增加入口。

### 5.3 记录表单

- 只保留金额、日期和备注。
- 金额输入使用独立白色卡片承载，其他字段保持轻量。
- 收入金额焦点使用蓝色，储蓄金额焦点使用橙色。
- 保存按钮全宽展示。
- 删除入口使用红色文字，最终操作需要二次确认。

### 5.4 趋势页

- 标题区与页面背景一致，不使用蓝色顶栏。
- 年度汇总为三列白色数据区域。
- 两张图表使用白色卡片，并同时显示横轴和纵轴。
- 图表卡片使用边框而非投影，坐标轴和图例字号不小于 11px。
- 收入柱使用蓝色，净储蓄柱和储蓄率折线使用橙色。
- 点击月份后展示该月精确数据。

### 5.5 设置页

- 使用简单的大标题和一个数据隐私设置列表。
- 设置页不提供储蓄计划和账户备注。
- “清空全部数据”仅使用红色文字，不增加“危险操作”标题。
- 点击清空后仍需输入“清空”进行二次确认。

### 5.6 备份与恢复

- 备份文件后缀为 `.backup`。
- 导出和恢复均使用密码。
- 恢复会覆盖现有数据，执行前必须明确确认。
- 页面明确提示所有业务数据仅保存在当前浏览器，不需要业务服务器；同时说明 Umami 仅记录匿名访问和关键功能使用情况。

## 6. 全局组件

### 全局导航

- MVP 不使用底部导航栏。
- 趋势和设置作为首页右上角的两个图标入口。
- 二级页面统一提供左上角返回入口。

### 按钮

- 收入主操作：深色按钮或蓝灰色描边按钮。
- 储蓄主操作：橙色底白色文字。
- 普通次操作：白底浅色边框。
- 危险确认：红色底白字。

### 底部弹层

- 顶部圆角 28px。
- 遮罩使用深灰蓝半透明色。
- 支持点击遮罩和 Esc 关闭非强制弹层。
- 取消操作在左，确认操作在右。

## 7. 状态与适配

- 空状态使用中性文案，不责备用户。
- 有收入但没有储蓄记录时使用橙色轻提示。
- 字段错误紧贴输入框展示。
- 支持 `prefers-reduced-motion`。
- 必须检查 320px、390px、430px 三个宽度。
- 大于 600px 时以最大 480px 的应用窗口居中展示。

## 8. 实现映射

- 色彩与尺寸变量：`src/styles/tokens.css`
- 全局与组件样式：`src/styles/global.css`
- 图标：`@phosphor-icons/react`
